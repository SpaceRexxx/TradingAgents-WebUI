## 核心规则(已确认)

**实值优先 + 估算必显式标记**:ETF 层面可取实值的指标必须给工具实值 + 数据截止日,不得用估算区间替代;底层指数派生量允许估算但必须显式标「估算·非工具直接读取」并写明推算依据;严禁把估算贴 `get_xxx` 来源标签伪装成实值。保留报告信息量,只杀伪装。

## fundamentals.md 改动

保留现有四节标题结构(数据源优先级 / 指标口径纪律 / QC 清单 / 常见错误),按下述调整内容并新增两节:

### 1. 重写"指标口径纪律"为硬规则
追加三条硬约束(在保留现有分组评估、趋势优先两条基础上):
- **实值优先**:ETF 层面可取实值的指标(总净资产/AUM、费用率、远期 P/E、股息率、前十持仓及权重等)**必须**给出 `get_fundamentals` 返回的具体数值 + 数据截止日;**严禁**用估算区间(如 `~24x–27x`)替代本可取到的实值。
- **估算必标**:纳指100底层加权派生量(加权 ROE、加权毛/净利率等)在工具无法直接取时**允许**估算,但**必须**显式标注「估算·非工具直接读取」并写明推算依据;**严禁**给估算贴 `get_xxx` 来源标签伪装成实值。
- **期间强制**:每个数据点必须带期间或数据截止日(对标 market.md 的 `(2026-05-18)`);无期间的数字一律视为未溯源,须按未溯源处理(标「未溯源」或补期间)。

### 2. 新增"强制输出汇总表"节
报告结尾必须包含一张汇总表,列固定为:`指标 | 数值 | 期间 | 来源(工具/报表) | 实值或估算`。最后一列只能填「实值」「估算」或「不适用」之一(「不适用」仅用于确无适用数据的情形,如 ETF 无传统三大报表)。

### 3. 新增"输出末尾自检"节(对标 market.md QC 块格式)
报告末尾必须追加固定格式自检块:
```
【自检】
☑ ETF 层面可取实值均给出实值+截止日,未用估算区间替代
☑ 底层派生估算均已显式标「估算」并说明推算依据
☑ 每个数据点带期间;无未溯源数字
☑ 估值/盈利/偿债现金流三组均覆盖;趋势优先于快照
```

### 4. QC 清单 / 常见错误各加一条
- QC 清单新增:`[ ] 实值与估算已区分,且估算未贴工具名伪装成来源`。
- 常见错误新增:`把本可取实值用估算区间替代;给估算贴工具名伪装成来源;数据点无期间`。
